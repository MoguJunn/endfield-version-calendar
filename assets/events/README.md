# 活动图示素材目录

把处理好的活动图放在此目录，并在 `app.js` 对应日程中增加：

```js
image: "./assets/events/op-wander.webp",
```

页面会自动把这张图用于时间轴活动条、下方日程卡片和详情弹窗；没有配置 `image` 时，会继续显示现有的风格化图示。

## 建议规格

- 格式：优先 WebP，保留透明背景的素材可使用 PNG。
- 建议尺寸：横图 `1200 × 400`；只拿到角色立绘时可制作 `800 × 800` 透明底版本。
- 单张体积：尽量控制在 250 KB 以内。
- 构图：主体放在图片右侧，左侧保留暗色或透明空间，避免覆盖活动名称。
- 命名：使用日程 `id`，例如 `op-wander.webp`、`sanity-supply-first.webp`。
- 来源：优先使用官方公告封面、官方角色立绘或官方活动视觉；记录原始链接和发布日期，不建议从当前低清海报直接裁图。

## 批量准备流程

1. 从官方公告保存无文字封面或角色立绘。
2. 用图片编辑工具裁成 3:1 横图，主体保持在右侧 40% 区域。
3. 叠加从透明到深色的左向渐变，确保白色日程文字可读。
4. 导出 WebP，并压缩到建议体积。
5. 在 `app.js` 对应事件对象加入 `image` 字段，运行 `npm run test` 和 `npm run build`。

请勿依赖第三方站点图片直链，避免防盗链、链接失效和来源不清晰。
