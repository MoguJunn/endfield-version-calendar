# 「向渊行」版本实时日历

依据官方「向渊行」版本内容时间轴整理的独立静态站点。页面按北京时间（UTC+8）实时计算活动状态、下一节点倒计时和当前时间线位置。

## 环境要求

- Node.js 20 或更高版本
- 无第三方运行时依赖，无需安装数据库或后端服务

## 本地运行

```powershell
npm run dev
```

打开 `http://127.0.0.1:4178`。

## 校验与构建

```powershell
npm run test
npm run build
```

构建产物位于 `dist`，可直接部署到任意静态网站托管平台。

## 维护活动数据

活动数据集中在 `app.js` 顶部的 `rawEvents` 数组。时间必须填写带 `+08:00` 时区的 ISO 8601 字符串；常驻开放项目的 `end` 使用 `null`。

卡池同期开放的签到和「作战演练」使用 `related` 合并显示在对应寻访条中，避免被重复统计为三项独立日程。武库申领等采用相对结束规则的项目使用 `endLabel`，不会被错误显示为固定日期；仅公布结束时间的项目使用 `startUnknown` 和 `startLabel` 明确标记开始时间待公布。

## 主站数据库同步

页面会读取主站公共接口 `https://ef-gacha.mogujun.icu/api/stats`：

1. 优先请求 `type=version_calendar`，加载当前启用的版本内容快照和规范卡池名称。
2. 新接口尚未部署或快照表尚未迁移时，回退请求 `type=pool_catalog`，仅同步角色池与武器池名称。
3. 网络不可用时使用 `app.js` 内置版本备份，页面不会因数据库故障而无法打开。

主站数据库迁移位于主站私有仓库的 `supabase/migrations/150_create_version_content_snapshots.sql`，不属于此前端仓库。部署到新的站点来源时，还需由主站维护者将该域名加入 `API_ALLOWED_ORIGINS`；本地 `localhost / 127.0.0.1` 不受此限制。

## 添加真实活动图示

站点已经支持真实图片和风格化占位图两级显示。把图片放入 `assets/events`，然后在对应活动数据中填写：

```js
image: "./assets/events/op-wander.webp",
```

图片会自动出现在时间轴、近期日程卡片和详情弹窗中。完整尺寸、压缩与构图规范见 `assets/events/README.md`。

页面内容依据版本日历图片人工整理。若活动名称或时间与游戏内不一致，请以官方公告和游戏内显示为准。

## 项目结构

```text
.
├─ index.html            # 页面结构
├─ styles.css            # 主题、布局与动效
├─ app.js                # 日程数据、实时状态和交互逻辑
├─ assets/events/        # 可选活动图片
└─ scripts/              # 本地服务器、构建和结构检查
```

`dist` 是由 `npm run build` 生成的部署产物，不纳入版本控制。

## 公开发布前说明

- 当前仓库未附带开源许可证；正式公开前需要由维护者选择许可证。
- `assets/events` 中只能加入有权公开再分发的图片，并保留来源和发布日期记录。
- 主站数据库、服务端密钥和部署配置不属于此前端仓库，禁止提交 `.env`、私钥或服务器连接文件。
- 项目名称、游戏名称和相关美术素材的权利归各自权利人所有；本项目是非官方日历工具。
